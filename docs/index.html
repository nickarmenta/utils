📊 Report Parser

Transform business reports into clean, segmented marketing lists

Overview

Report Parser is a powerful Python tool that processes spreadsheet exports from business systems and transforms them into clean, validated, and segmented contact lists ready for marketing campaigns.

View on GitHub Get Started

Key Features

🧹 Data Cleaning

  • Remove database UUIDs/IDs
  • Merge phone numbers
  • Parse addresses into components
  • Validate location data

🗺️ Regional Segmentation

  • Split contacts by geography
  • Three mailing list regions
  • Automatic state filtering
  • Separate exports per region

⚡ Dual Interface

  • Command-line interface (CLI)
  • Interactive web app (Streamlit)
  • File upload or folder processing
  • Real-time preview

Mailing List Regions

Installation

Using uv (recommended)

uv sync

Using pip

pip install -e .

Configuration

Create a .env file in the project root: 

BASE_DIRECTORY=C:\Path\To\Your\Downloads

Usage

💻 Command Line Interface 

# Process contacts (single file)
uv run python -m utils contacts

# Process contacts with regional segmentation
uv run python -m utils contacts --segment

# Process visits
uv run python -m utils visits

🌐 Web Interface

# Launch Streamlit app
uv run streamlit run app.py

The web app provides:

  • Interactive file upload (no need for Downloads folder)
  • Toggle for regional segmentation
  • Preview of processed data
  • Individual download buttons for each region

Contact Processing Pipeline

Step Action Result
1. Clean IDs Remove all UUID/ID columns Cleaner dataset
2. Merge Phones Combine Phone & Mobile fields Single phone column
3. Parse Address Extract City, State, Zip from Business Address 3 separate location columns
4. Validate Data Check City (no numbers), State (2 letters), Zip (5 digits) Clear invalid entries
5. Filter States Keep only 15 allowed states Targeted geography
6. Segment (optional) Split into PNW, CA, GG regions 3 separate mailing lists

Output

Without Segmentation

Single file: Contact_Processed_[timestamp].xlsx

With Segmentation

Three files:

  • Contact_PNW_[timestamp].xlsx
  • Contact_CA_[timestamp].xlsx
  • Contact_GG_[timestamp].xlsx

Columns in Output

Name Account Name Email Phone City State Zip

Data Validation Rules

City Validation

  • Must be more than 2 characters
  • Cannot contain numbers
  • Invalid entries are cleared

State Validation

  • Must be exactly 2 characters
  • Must be alphabetic only
  • Must be in allowed list

Zip Code Validation

  • Must start with 5 digits
  • Extended ZIP+4 format allowed
  • Invalid entries are cleared
